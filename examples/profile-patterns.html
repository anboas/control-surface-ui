Implementation notes

Reusable profile contract

Use these rules when adding a new top-right profile area.

  • Use .if-account-menu for identity triggers and .if-popover__panel.if-account-surface for rich account panels.
  • Use .if-menu only when the dropdown is a short command list and does not need identity, preferences, or metrics.
  • Keep destructive or security-sensitive actions visually separated in their own account-surface section.
  • Expose current user, active workspace, session status, and role state before the user has to click when those facts change behavior.
  • Theme and density controls should reuse the same data-if-theme, data-if-theme-label, button, badge, and field primitives used elsewhere.
Minimum markup <div class="if-popover if-account-popover" data-if-popover>...<button class="if-account-menu" data-if-popover-toggle="account-menu">...</button><section class="if-popover__panel if-account-surface" id="account-menu" hidden>...</section></div>
